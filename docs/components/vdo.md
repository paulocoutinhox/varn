# 🗄️ vdo

VDO (Varn Data Objects) is a PDO-style database layer. One API connects to SQLite, MySQL/MariaDB,
and PostgreSQL — the driver is chosen from the DSN scheme. It is written in Lua on top of the native
`ffi` module and loads the matching client library on connect, so a backend you never use is never
loaded.

Add the components directory to `package.path` first (see [components](../components.md)):

```lua
package.path = package.path .. ";./components/?.lua;./components/?/init.lua"
local vdo = require("vdo")
```

The native client library must be installed on the host: `sqlite3` for SQLite, `libmysqlclient` or
`libmariadb` for MySQL/MariaDB, and `libpq` for PostgreSQL.

## Connecting

`vdo.connect(dsn, username?, password?, options?)` → a connection. The DSN scheme selects the driver:

| Scheme | DSN example |
|--------|-------------|
| `sqlite` | `sqlite::memory:` · `sqlite:/var/data/app.db` · `sqlite:app.db` |
| `mysql` / `mariadb` | `mysql:host=127.0.0.1;port=3306;dbname=app` |
| `pgsql` / `postgres` | `pgsql:host=127.0.0.1;port=5432;dbname=app` |

```lua
local db = vdo.connect("mysql:host=127.0.0.1;dbname=app", "user", "secret")
```

SQLite keeps the remainder of the DSN as the path. MySQL and PostgreSQL parse it as
`key=value;key=value` pairs (`host`, `port`, `dbname`, `unix_socket`, `charset`, …); `username` and
`password` are passed separately, like PDO.

Errors are raised, matching PDO's exception mode. Wrap a call in `pcall` to handle a failure.

## Connection

- `conn:exec(sql)` → runs a literal statement and returns the affected row count. Use it for DDL and
  for writes without bound parameters.
- `conn:query(sql, params?)` → runs a statement and returns a [Statement](#statement) ready to fetch.
- `conn:prepare(sql)` → compiles a [Statement](#statement) you can execute one or more times.
- `conn:lastInsertId(sequence?)` → the id generated by the last insert. PostgreSQL accepts an
  optional sequence name and otherwise reports `lastval()`.
- `conn:beginTransaction()` · `conn:commit()` · `conn:rollBack()` → transaction control.
- `conn:inTransaction()` → `true` while a transaction is open.
- `conn:close()` → closes the connection.

## Statement

- `stmt:execute(params?)` → runs (or re-runs) the statement and returns the statement itself, so it
  chains with a fetch. Pass `params` as an array for `?` placeholders or a map for `:name` ones.
- `stmt:fetch()` → the next row as a table keyed by column name, or `nil` when the rows are exhausted.
- `stmt:fetchAll()` → every remaining row as an array of tables.
- `stmt:rowCount()` → the number of rows **changed** by a write, or **retrieved** by a select. SQLite
  reports `0` for a select, matching PDO.
- `stmt:columnCount()` → the number of columns in the result.
- `stmt:close()` → releases the statement.

Values map to Lua by column type: integers and floats become numbers, booleans become booleans
(PostgreSQL), `NULL` becomes `nil`, and everything else is returned as a string.

## Placeholders

Both positional and named placeholders are supported, and the parser ignores anything inside string
literals, line comments, or PostgreSQL `::` casts:

```lua
-- positional: bind with an array
local s = db:prepare("SELECT * FROM users WHERE age >= ? AND city = ?")
s:execute({ 18, "Lisbon" })

-- named: bind with a map
local s = db:prepare("INSERT INTO users (name, score) VALUES (:name, :score)")
s:execute({ name = "alice", score = 9.5 })
```

For SQLite the placeholders bind natively, for PostgreSQL they are rewritten to `$1..$n`, and for
MySQL they are bound through `mysql_real_escape_string`, matching PDO's default emulated prepares.

## Examples

### `crud.lua`

```lua
-- driver-agnostic crud: defaults to in-memory sqlite, set VDO_DSN to target another backend.
local dir = arg[0]:match("^(.*)[/\\]") or "."
package.path = ("%s/../../?.lua;%s/../../?/init.lua;"):format(dir, dir) .. package.path

local vdo = require("vdo")

local db = vdo.connect(os.getenv("VDO_DSN") or "sqlite::memory:", os.getenv("VDO_USER"), os.getenv("VDO_PASS"))

db:exec("DROP TABLE IF EXISTS users")
db:exec("CREATE TABLE users (id INTEGER PRIMARY KEY, name TEXT NOT NULL, score REAL NOT NULL)")

local insert = db:prepare("INSERT INTO users (name, score) VALUES (:name, :score)")
insert:execute({ name = "alice", score = 9.5 })
insert:execute({ name = "bob", score = 7.25 })
insert:execute({ name = "carol", score = 8.0 })
insert:close()

print("last insert id:", db:lastInsertId())

local query = db:prepare("SELECT id, name, score FROM users WHERE score >= ? ORDER BY score DESC")
query:execute({ 8.0 })
for _, row in ipairs(query:fetchAll()) do
    print(string.format("user %d: %-6s %.2f", row.id, row.name, row.score))
end
query:close()

db:beginTransaction()
local bump = db:prepare("UPDATE users SET score = score + 0.5")
bump:execute({})
print("rows updated:", bump:rowCount())
bump:close()
db:commit()

local total = db:query("SELECT count(*) AS n, avg(score) AS avg FROM users"):fetch()
print(string.format("rows: %d  average: %.3f", total.n, total.avg))

db:close()
print("vdo crud ok")
os.exit(0)
```

### `prepared.lua`

```lua
-- one prepared statement reused across executions, with row-at-a-time fetching.
local dir = arg[0]:match("^(.*)[/\\]") or "."
package.path = ("%s/../../?.lua;%s/../../?/init.lua;"):format(dir, dir) .. package.path

local vdo = require("vdo")

local db = vdo.connect(os.getenv("VDO_DSN") or "sqlite::memory:", os.getenv("VDO_USER"), os.getenv("VDO_PASS"))

db:exec("DROP TABLE IF EXISTS events")
db:exec("CREATE TABLE events (id INTEGER PRIMARY KEY, kind TEXT NOT NULL, value INTEGER NOT NULL)")

local insert = db:prepare("INSERT INTO events (kind, value) VALUES (?, ?)")
for i = 1, 6 do
    insert:execute({ i % 2 == 0 and "even" or "odd", i })
end
insert:close()

local byKind = db:prepare("SELECT value FROM events WHERE kind = ? ORDER BY value")
for _, kind in ipairs({ "odd", "even" }) do
    byKind:execute({ kind })

    local values = {}
    while true do
        local row = byKind:fetch()
        if not row then
            break
        end
        values[#values + 1] = row.value
    end

    print(kind, table.concat(values, ", "))
end
byKind:close()

db:close()
print("vdo prepared ok")
os.exit(0)
```
